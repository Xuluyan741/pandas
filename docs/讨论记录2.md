### 二、重点结论汇总

- **消息推送（Web Push）当前状态**：未配置  
  - 缺少 `NEXT_PUBLIC_VAPID_PUBLIC_KEY` 和 `VAPID_PRIVATE_KEY` 两个环境变量  
  - `isPushConfigured()` 返回 `false`，导致 `/api/push/vapid-public` 和各类 `/api/cron/*` 推送逻辑会走「未配置」分支

- **启用推送需要的关键步骤**：
  1. 使用 `npx web-push generate-vapid-keys` 生成 VAPID 公私钥  
  2. 在 `.env.local` 或部署环境中设置：  
     - `NEXT_PUBLIC_VAPID_PUBLIC_KEY=<生成的 public key>`  
     - `VAPID_PRIVATE_KEY=<生成的 private key>`  
  3. 生成一个随机的 `CRON_SECRET`（如 `openssl rand -base64 24`），同时在：  
     - 本地 `.env.local`  
     - 部署平台环境变量（如 Vercel）  
     中写入，保证 `/api/cron/daily-push`、`/api/cron/agent-push`、`/api/cron/heartbeat` 等能通过鉴权

- **其它与推送相关的要点**：
  - 用户需要在浏览器允许通知，并至少打开一次首页，前端才会注册 Push Subscription  
  - 定时任务由 Vercel Cron 或外部 Cron 调用 `/api/cron/*` 路由实现

### 三、配置项清单（按功能分组）

#### 1. 基础认证与站点

- `NEXTAUTH_SECRET`：NextAuth 用的加密密钥（必须为随机安全字符串）
- `NEXTAUTH_URL`：站点地址（开发：`http://localhost:3000`；生产为实际域名）

#### 2. Google 登录（可选）

- `GOOGLE_CLIENT_ID`：Google OAuth Client ID
- `GOOGLE_CLIENT_SECRET`：Google OAuth Client Secret
- `NEXT_PUBLIC_GOOGLE_ENABLED`：是否在前端显示 Google 登录按钮（`"true"` / `"false"`）

#### 3. 数据库（Turso，可选但生产推荐）

- `TURSO_DATABASE_URL`：Turso 数据库地址
- `TURSO_AUTH_TOKEN`：Turso 访问 token

不配置时，项目会退回本地 `data/app.db` 的 SQLite。

#### 4. 每日推送 / Agent 推送 / 心跳推送

- `NEXT_PUBLIC_VAPID_PUBLIC_KEY`：Web Push VAPID 公钥（前端可见）
- `VAPID_PRIVATE_KEY`：Web Push VAPID 私钥（仅服务端）
- `CRON_SECRET`：保护 `/api/cron/*` 的密钥，定时任务调用接口时需携带

#### 5. AI 相关

- `DEEPSEEK_API_KEY`、`DEEPSEEK_MODEL`、`DEEPSEEK_BASE_URL`：DeepSeek 文本解析与 Agent 能力
- `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_TRANSCRIBE_MODEL`：语音转文字、多模型对话
- `ANTHROPIC_API_KEY`：Claude，用于深度推理
- `GOOGLE_API_KEY`：Gemini，用于意图分类与语音
- `VLLM_API_BASE`、`VLLM_API_KEY`、`VLLM_MODEL`：vLLM / 自建模型

#### 6. 扩展能力与通道

- `MCP_SERVER_URL`、`MCP_SERVER_HEADERS`：MCP 服务配置
- `SERPER_API_KEY`、`BRAVE_API_KEY`：搜索引擎 API（长期目标资料 / web_search）
- `TELEGRAM_BOT_TOKEN`：Telegram 机器人

#### 7. Agent 工作区与安全

- `AGENT_WORKSPACE_DIR`：Agent 工作区根目录
- `AGENT_RESTRICT_TO_WORKSPACE`：是否限制访问工作区之外
- `AGENT_SHELL_ENABLED`：是否允许 Agent 使用 Shell
