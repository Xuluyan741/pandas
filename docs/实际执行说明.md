# 实际执行说明：说了什么会真的发生什么

> 避免「做了很多却感觉什么都没执行」——这里明确：哪些会**自动执行**，哪些**必须再点一步/配好环境**才执行。

---

## 一、主对话里「说了就真的会执行」的

| 你说的话 | 实际会发生什么 | 条件 |
|----------|----------------|------|
| 「明天下午三点开会」且**无时间冲突** | 前端会**直接**调用 `addTask`，日程里**立刻多一条任务**，无需你再点 | 有至少一个项目（没有会提示先建项目） |
| 「搜一下 XXX」 | 后端调 Serper/Brave，回复里**直接追加**「【网页搜索】」结果 | `.env` 里配了 `SERPER_API_KEY` 或 `BRAVE_API_KEY` |
| 「五一去韩国旅游」等长期目标 | 后端**真的**搜资料 + 生成计划，返回清单和参考资料 | 无额外条件（搜资料更好需 `SERPER_API_KEY`） |
| 打车/订餐/买票等 | 生成 Deep Link 卡片，你点「允许并打开」**真的**跳转外部 App/网页 | 无 |

所以：**日程写入**在「无冲突」时是自动的；**搜索**在配好 key 后是自动的；**长期目标**的规划和搜资料是自动的，只是「写入日历」要你再点一下按钮。

---

## 二、需要「你再点一步」才执行的（设计如此）

| 能力 | 为什么不是全自动 |
|------|------------------|
| 解析出任务但有**时间冲突** | 需要你选「好的」按建议改，或「暂不」保留原样，再写入 |
| 长期目标的「写入日历」 | 计划可能很多条，需要你确认后再点「一键写入到日历」 |
| Deep Link（打车/订餐等） | 涉及跳转/支付，必须你点「允许并打开」才跳转 |

这些不是没执行，而是**故意**要你确认后再执行。

---

## 三、需要「先配置或先安装」才执行的

| 能力 | 要满足什么才「真的执行」 |
|------|--------------------------|
| **网页搜索** | `.env` 里 `SERPER_API_KEY` 或 `BRAVE_API_KEY`，否则只提示「暂时无法联网搜索」 |
| **社区技能（ClawHub）发现/安装/执行** | 1）`CLAWHUB_API_BASE` 不设为 `"0"`；2）登录；3）安装过的技能才会在本轮「执行」 |
| **MCP 工具调用** | 主界面**没有**接 MCP 对话入口，目前只有配好 `MCP_SERVER_URL` 或从商店安装后，**单独调** `POST /api/agent/mcp/chat` 才会执行 MCP 工具 |

所以：感觉「没执行」时，多半是这里某一条没满足（没配 key、没登录、没安装技能、或 MCP 没在主界面用上）。

---

## 四、怎么自测「确实在执行」

1. **日程**：说「明天上午写周报」→ 看工作台/任务列表是否多一条；若无，看是否报了冲突（有冲突就点「好的」再看）。
2. **搜索**：配好 Serper 或 Brave key 后说「搜一下 项目管理工具」→ 回复里应出现「【网页搜索】」和几条结果。
3. **长期目标**：说「帮我规划五一去韩国旅游」→ 应出现计划清单 + 参考资料；点「一键写入到日历」后，任务列表里应有对应任务。

---

## 五、小结

- **会直接执行的**：无冲突时的任务写入、配好 key 后的网页搜索、长期目标的规划与搜资料、Deep Link 的生成（跳转需你点一下）。
- **要你点一下才执行的**：有冲突时的任务写入、长期目标写入日历、Deep Link 跳转。
- **要你先配置/登录/安装才执行的**：搜索（key）、社区技能（ClawHub + 登录 + 安装）、MCP（配置或商店安装，且主界面尚未接 MCP 对话）。

若你希望某一步「说了就执行、完全不用点」，可以具体说哪一步（例如「无冲突时静默写入并 toaster 提示」或「长期目标也自动写入」），我们可以按产品意图改成默认行为或加开关。
