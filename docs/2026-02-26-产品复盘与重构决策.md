## 一、PRD 深度复盘（五位专家视角）

### 专家一：商业战略（前 Notion/Todoist 视角）

- **核心问题**：产品定位过于发散，同时想做项目管理、灵魂伴侣、生活管家、抢票助手、财务中枢
- **建议**：一句话说清用户为什么用你而不是滴答清单 + ChatGPT
- **商业模式缺失**：PRD 全文未提具体 monetization 方案，需明确免费/付费边界
- **"灵魂伴侣"定位风险**：日程管理用户心智是"效率工具"，强行嫁接情绪陪伴可能两头不讨好
- **缺少用户增长飞轮**：无病毒传播机制

### 专家二：技术架构（前 Google Calendar 视角）

- **多级模型路由是伪需求**：MVP 阶段用 5 种模型 + 路由层是巨大运维负担，一个 DeepSeek 就够
- **四层架构过早抽象**：感知层/大脑层/动作层/记忆层是论文架构，先做 API Route → LLM → JSON
- **冲突消解不需要 LLM**：时间重叠检测是纯算法问题，LLM 只负责模糊输入→结构化和建议文案
- **CDN 加载 frappe-gantt 是技术债**
- **缺少离线优先策略**：Zustand ↔ Turso 同步冲突未设计

### 专家三：用户体验（前 Things 3/Fantastical 视角）

- **技能树太复杂**：9 个模块用户认知负担巨大，建议砍到 3 个核心功能
- **缺少用户旅程图**：描述了"能做什么"但没描述"用户怎么用"
- **推送滥用风险**：需日推送上限 ≤ 3 条 + 退避策略
- **没有空状态设计**：新用户 0 任务时看到什么决定 Day 1 留存

### 专家四：AI 产品（前 Anthropic/OpenAI 视角）

- **自然语言解析准确率是生死线**：AI 解析错误 → 手动修改 → 比直接录入还慢 → 流失
- **长期目标管家的 AI 幻觉风险极高**：搜寻资料可能失效/误导
- **缺少 AI 成本预算机制**：免费用户疯狂对话的成本谁承担
- **"零配置主动代理"是 2028 年的事**：不应影响当前设计决策

### 专家五：市场竞争（前 YC 视角）

- **竞品分析完全缺失**：Motion（AI 自动排程，融资 $30M+）、滴答清单、Notion Calendar 均未提及
- **7 个 Phase 计划太长**：创业产品应 2 周一个迭代，只规划到 Phase 2

## 二、护城河分析

### 产品负责人与 Gemini 讨论的四个护城河方向

1. **数据护城河**：用户长周期记忆与画像（用了一个月后不可替代）
2. **执行护城河**：深度集成私有 API（Apple Health、Google Calendar 等），迁移成本高
3. **垂直工作流**：针对备考/减肥等场景打磨极细的 SOP + 搜索源筛选
4. **主动式情感化监督**：个性化督导算法 + 品牌认同

### AI 顾问的判断

- 方向正确，但**当前一个护城河都还没有**
- 护城河不是规划出来的，是用户用出来的
- **第一道护城河 = 让 100 个用户每天打开你的产品**

## 五、关键决策记录

### 决策 1：品牌更名

- ~~小龙虾智能管家~~ → **小熊猫**（Red Panda）
- 理由：熊猫天然具有情绪价值、亲和力，比龙虾更适合"陪伴"定位

### 决策 2：界面重构方向

- **主界面**：以 AI 对话框为核心（类似 Spotlight / Raycast 的中心输入框）
- **现有功能**（任务卡、进度条、甘特图、倒计时等）：隐藏到悬浮球中
- **设计哲学**：功能都在，但不焦虑。用户看到的是对话，不是待办清单
- **参考**：Things 3 的极简、Notion AI 的对话优先、Apple Spotlight 的聚焦

### 决策 3：开发优先级

| 顺序 | 任务 | 预计时间 |
|------|------|---------|
| 1 | 界面重构（对话优先 + 悬浮球） | 本周 |
| 2 | 冲突消解引擎 | 下周 |
| 3 | 推送智能化（Heartbeat 模式） | 第三周 |

### 决策 4：技术精简

- Phase 1-3 只用一个 LLM（DeepSeek 或 Claude Sonnet）
- 冲突检测用纯算法，LLM 只负责自然语言→JSON 和建议文案
- 不引入 Vector DB / Graph DB / LangGraph，SQLite 够用
- 不做搜索源筛选、MultiOn、Apple Health 等远期功能

## 六、核心原则

1. **砍掉 80% 的规划，聚焦 20% 的执行**
2. **停止讨论架构，开始写代码**
3. **每做一个功能，交互就是最终形态**（不做丑的再回来改）
4. **一英寸宽，一英里深**：找到极窄场景，体验做到极致
