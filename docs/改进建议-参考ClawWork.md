# 改进建议：参考 ClawWork 项目

> [ClawWork](https://github.com/HKUDS/ClawWork) 是 HKUDS 的「AI 同事」经济基准：Agent 用 $10 起步、为每次 token 付费、通过完成真实职业任务赚取收入，多模型同台竞技，React 看板实时展示余额与生存指标。本文对照小熊猫现状提出可落地的改进方向。

---

## 一、ClawWork 核心机制速览

| 维度 | ClawWork | 小熊猫现状 |
|------|----------|------------|
| 经济压力 | 初始 $10，每 token 扣费，收入仅来自完成任务 | 按**次数**配额（日/月），无 token 级计费与展示 |
| 任务来源 | GDPVal 220 任务、44 行业、BLS 时薪定价 | 用户日程/长期目标，无「任务价值」量化 |
| 产出评估 | LLM 评估质量 → 按质量×工时×时薪付酬 | 无完成质量评估，仅有冲突消解建议与计划生成 |
| 决策结构 | decide_activity(work / learn)，工作 vs 学习取舍 | 无显式「工作 vs 学习」决策层 |
| 看板 | React + WebSocket：余额曲线、任务完成、学习时间线 | 大盘+日历+对话，无 AI 使用/成本可视化 |
| 成本追踪 | 每任务 token_costs.jsonl、balance_after | providers 内部算 costUSD，未持久化、未对用户展示 |

---

## 二、改进建议（按优先级）

### 1. 成本与配额可视化（高价值、可先做）

**ClawWork 做法**：Dashboard 实时展示 Balance、Income、Cost、Survival status；每任务有 token_costs 记录。

**小熊猫可做**：

- 在**设置或单独「使用统计」页**增加：
  - 本日/本月已用次数（按 `agent_chat` / `parse_tasks` / `scheduler` / `agent_push` 分类型）
  - 剩余额度（日/月）
  - （可选）等效成本：从现有 `providers.ts` 的 `costUSD` 汇总到 `ai_usage` 或新表 `ai_costs`，按日/月展示「约消耗 ¥x」或「约 x 次调用」
- **后端**：在 `recordUsage` 时若已有 `costUSD`（如从 unified 返回），可写入 `ai_costs(user_id, period, kind, count, cost_usd)`，便于图表与 Pro 计费细化。

**效果**：用户有「经济感」、更清楚 Pro 的价值，且为后续「按价值计费」打基础。

---

### 2. 按「价值」区分配额消耗与奖励（中高）

**ClawWork 做法**：任务有明确 payment 区间（质量×工时×时薪）；完成高质量工作才增收。

**小熊猫可做**：

- **消耗**：不同 `UsageKind` 赋予不同「权重」（例如 `agent_chat`=1，`scheduler`=0.5，`agent_push`=0.3），配额按「加权次数」扣减，避免「一次长对话」与「一次推送」等价。
- **奖励**：用户完成高价值动作时返还额度或增加「成就点数」：
  - 例如：接受冲突消解建议并应用 → 返还 0.5 次；完成长期目标里程碑（勾选子任务）→ 返还 1 次或解锁一次免费规划。
- 仍保持「次数」为主、不强行改成「虚拟货币」，避免与 ClawWork 的完整经济模型混淆；重点是**让高价值行为被识别并给予正向反馈**。

**效果**：鼓励「真正用冲突消解、长期目标」而不是纯闲聊，与产品定位一致。

---

### 3. 「工作 vs 学习」的轻量抽象（中）

**ClawWork 做法**：`decide_activity(work | learn)`；Agent 每天选择是立刻干活赚钱还是花时间学习以提升后续表现。

**小熊猫可做**：

- 不引入完整 work/learn 循环，而是做**一次决策提示**：
  - 在 Agent 推送或每日摘要时，由 LLM 给出「今日建议」：优先「执行日程/消解冲突」还是「花 5 分钟做长期目标拆解/复盘」。
  - 前端可展示为：「小熊猫建议：今天先处理 3 条冲突，再花一点时间更新考研计划。」
- 或仅在**长期目标**模块中区分「执行」（勾选子任务）与「规划」（生成/调整计划），在统计中区分「执行步数」与「规划步数」，用于后续「奖励额度」或成就。

**效果**：用户感知到「小熊猫在帮自己做优先级判断」，而不只是被动响应。

---

### 4. 任务完成质量/价值评估（中、可选）

**ClawWork 做法**：提交 work 后由 LLM（如 GPT-5.2）按行业 rubric 打分，质量分参与报酬计算。

**小熊猫可做**：

- **轻量版**：不对单次对话做 LLM 评估，只做**行为统计**：
  - 用户点击「接受建议」次数、冲突消解采纳率、长期目标子任务完成率等，作为「质量」的代理指标。
- **完整版**（若后续有 Pro 差异化）：对「冲突消解结果」或「生成的计划」做一次轻量 LLM 评估（1 条 prompt，低 token），输出 0–1 或「高/中/低」价值，用于：
  - 成就系统、「本周最有价值操作」回顾；
  - 或与「奖励额度」挂钩（高质量完成 → 返还次数）。

**效果**：从「做了多少」升级到「做得多好」，与 ClawWork 的 production 导向一致。

---

### 5. 多模型/策略对比与路由优化（中低）

**ClawWork 做法**：多模型（GLM、Kimi、Qwen、Claude、Gemini）同台竞技，看谁在相同任务下赚得多、存活久。

**小熊猫可做**：

- 已有**意图路由**（TRIVIAL/EXTRACTION/REASONING 等），可扩展为：
  - 对同一意图的请求，按比例 A/B 到不同模型（如 90% DeepSeek / 10% Claude），并记录该次调用的 `model`、`costUSD`、以及用户后续行为（是否接受建议、是否继续追问）。
  - 定期汇总：按意图×模型看「成本 vs 采纳率」，用于调整 DEFAULT_ROUTE_MAP。
- 不需要做「多 Agent 竞技看板」，只需**埋点 + 内部分析**即可。

**效果**：在控制成本的前提下，用真实使用数据优化模型选择，而不是纯主观配置。

---

### 6. 持久化「学习」与策略复用（中低）

**ClawWork 做法**：`learn(topic, knowledge)` 写入持久记忆，供后续决策复用。

**小熊猫可做**：

- 已有 `agent_memory`（key-value），可**约定 key 规范**：
  - 例如从冲突消解中提炼：「用户常接受“挪到周四”类建议」→ 写入 `preference:reschedule_accept` 或 `strategy:conflict_often_accept_move`。
  - 下次生成建议时，在 system prompt 中注入「用户历史偏好：更接受调整时间而非删除」。
- 或从长期目标完成情况中归纳：「用户对「考试」类目标完成率高于「健身」」→ 推送时优先提醒考试相关子任务。

**效果**：越用越「懂」用户，减少无效建议，贴近 ClawWork 的 learn 积累。

---

### 7. 产出物（Artifact）可追溯（低、可选）

**ClawWork 做法**：`submit_work` 提交文档/文件路径，作为可评估的交付物。

**小熊猫可做**：

- 将「对话轮次中产生的结构化结果」存为轻量 artifact：
  - 例如：本次对话生成的计划（长期目标）、本次冲突消解的建议摘要、本次生成的周报草稿。
  - 存为 `artifacts(user_id, type, ref_id, summary, created_at)` 或仅在现有表中加 `artifact_summary` 字段。
- 用于：用户「历史建议」回顾、或后续做「完成质量」统计时关联到具体产出。

**效果**：不改变主流程，但为「价值评估」和「回顾」留扩展点。

---

### 8. Dashboard 与实时感（低、偏展示）

**ClawWork 做法**：React 看板 + WebSocket 实时更新余额与任务状态。

**小熊猫可做**：

- 若已做「使用统计」页，可增加简单**趋势图**（按日/周的使用次数或等效成本），无需 WebSocket，用服务端聚合即可。
- 若有「小熊猫状态」面板，可在对话过程中短暂展示「本句消耗约 x 次」或「今日剩余 x 次」（已有 `canConsume` 数据），增强「经济感」而不必做完整经济系统。

**效果**：用户对配额与成本有直观感知，与 ClawWork 的「透明经济」理念一致。

## 三、实施顺序建议

| 优先级 | 建议项 | 依赖 | 预估 |
|--------|--------|------|------|
| P0 | 成本与配额可视化（统计页 + 可选 cost 落库） | 无 | 1–2 天 |
| P1 | 按价值区分消耗/奖励（加权次数 + 高价值行为返还） | quota 扩展 | 1–2 天 |
| P2 | 工作 vs 学习轻量抽象（推送/摘要中的「今日建议」） | 现有 agent-push | 0.5–1 天 |
| P2 | 持久化学习与策略复用（memory key 规范 + prompt 注入） | agent_memory | 1 天 |
| P3 | 任务完成质量/行为统计（采纳率、完成率） | 前端埋点 + 简单聚合 | 1 天 |
| P3 | 多模型 A/B 与路由优化（埋点 + 分析） | 现有 router | 1 天 |
| P4 | Artifact 可追溯、Dashboard 趋势图 | 可选 | 按需 |

## 四、与 ClawWork 的差异说明

- **小熊猫**定位是**智能日程管家**，不是「用经济生存基准测模型」；因此不引入真实货币、不做了就「死亡」的 balance，也不接 GDPVal 任务集。
- 借鉴的是：**成本可见、价值可感、决策可解释、学习可积累**；用「配额 + 加权 + 奖励 + 统计」在 C 端产品里做出轻量版「经济感」，而不做完整经济仿真。
