## 二、重点结论与决策记录

### 2.1 环境变量与平台

- **最小运行必需**  
  - `DEEPSEEK_API_KEY`：必须配置，否则对话/解析/冲突/长期目标全部不可用。
- **想「全部功能都用」时需要额外配置的核心变量**（详见 `docs/功能测试清单.md` 1.1 小节）：  
  - 登录：`NEXTAUTH_SECRET`、`NEXTAUTH_URL`；可选 Google 登录 `GOOGLE_CLIENT_ID`、`GOOGLE_CLIENT_SECRET`、`NEXT_PUBLIC_GOOGLE_ENABLED=true`  
  - 网页搜索：`SERPER_API_KEY` 或 `BRAVE_API_KEY`（二选一即可）。  
  - 社区技能：`CLAWHUB_API_BASE` 保持为默认 `https://clawhub.ai`（不要设为 `0`）。  
  - MCP：`MCP_SERVER_URL`（可选 `MCP_SERVER_HEADERS`），配好后主对话自动按意图调用 MCP 工具，并在回复中追加【MCP】块。  
  - 推送：`NEXT_PUBLIC_VAPID_PUBLIC_KEY`、`VAPID_PRIVATE_KEY`、`CRON_SECRET`。  
  - 云端数据库：`TURSO_DATABASE_URL`、`TURSO_AUTH_TOKEN`（登录后数据持久化到 Turso）。  
  - 支付：Stripe 一组变量，用于 Pro 订阅（本地测试可选配）。
- **平台链接汇总** 已写入 `docs/功能测试清单.md` 的「1.2 平台与链接汇总」，涵盖 DeepSeek、Serper、Brave、Google Console、Anthropic、OpenAI、Turso、Stripe、MCP Registry 等。

### 2.2 功能行为与「实际执行」

关键结论已整理在 `docs/实际执行说明.md`，要点如下：

- **自动执行（无需再点）**  
  - 无冲突的日程添加：「明天上午写周报」这类，会直接写入任务列表。  
  - 搜索：配置了 `SERPER_API_KEY` 或 `BRAVE_API_KEY` 后，说「搜一下 XXX」会直接追加【网页搜索】结果。  
  - 长期目标规划：如「帮我规划五一去韩国旅游」，会自动「识别长期目标 → 搜资料 → 生成分步计划」并展示卡片。  
  - Deep Link 卡片生成：打车/订餐/买票等会生成可点开的 Deep Link，但真正跳转需用户确认。
- **需用户确认后执行**  
  - 冲突任务：检测到冲突时，需点「好的」或「暂不」后才写入/调整。  
  - 长期目标写入日历：点卡片上的「一键写入到日历」后才创建对应子任务。  
  - Deep Link 跳转：必须点「允许并打开」才会打开外部 App 或网页。
- **需额外配置或安装才会执行**  
  - 网页搜索、社区技能（ClawHub）、MCP，在未配置相应 key/URL 时仅有占位或提示，不会真正联网/调用工具。

### 2.3 MCP、多 MCP 与 MCP 商店

- **多 MCP 配置**  
  - 新增 `user_mcp_servers` 表与 `src/lib/mcp-config.ts`，支持：  
    1. `.env` 中的 `MCP_SERVER_URL`（单条）；  
    2. `.env` 中的 `MCP_SERVERS`（JSON 多条）；  
    3. 用户安装记录（`user_mcp_servers`）。  
  - `/api/agent/mcp/chat` 与 `/api/agent/mcp/tools*` 会合并所有 MCP 服务器，将工具名加前缀区分来源。
- **MCP 商店 API**  
  - `GET /api/mcp/store`：返回推荐 MCP 服务器列表（静态，可后续接官方 Registry）。  
  - `GET /api/mcp/installed`：查看当前用户已安装的 MCP。  
  - `POST /api/mcp/install` / `/api/mcp/uninstall`：安装/卸载 MCP 服务器。
- **主对话内直接使用 MCP**  
  - 在 `agent/chat` 中新增 `runMcpRoundForMainChat`，若配置了 MCP 且当前输入与 MCP 工具匹配，会自动触发一轮 MCP 调用，将结果以【MCP】块追加到同一条回复中。

### 2.4 社区技能（ClawHub）安装与执行

- 新增表 `community_skills_installed` 与 API：  
  - `GET /api/skills/community/installed`、`POST /api/skills/community/install`、`POST /api/skills/community/uninstall`。  
- 在 `agent/chat` 中：  
  - 发现相关 ClawHub 技能 → 若已安装则执行 `runCommunitySkill`（按 SKILL.md + LLM）；  
  - 若未安装则在前端显示「推荐安装」卡片，用户点「安装」后写入 DB，下次对话可自动执行。

### 2.5 多模型路由与错误修复

- 引入多模型路由（Gemini Flash / Claude / GPT o3-mini / DeepSeek / vLLM），根据意图选择模型。  
- 修复了只配置 DeepSeek 时，ACTION 场景落到 Anthropic 抛出 **`ANTHROPIC_API_KEY 未配置`** 的问题：  
  - 在 `callWithFallback` 中，当主模型和 fallback provider 未配置时，统一回退到 `MODELS.deepseekChat`，确保只配 `DEEPSEEK_API_KEY` 也能正常工作。

### 2.6 NextAuth JWT 解密错误处理

- 在 `src/app/layout.tsx` 中对 `getServerSession(authOptions)` 增加 try/catch：  
  - 当 `NEXTAUTH_SECRET` 变更或旧 session cookie 无法解密时，不再阻塞渲染，而是记录 warning 并视为未登录状态。  
  - 用户可通过清空浏览器 cookie 或无痕窗口重新登录解决根因。
