## 三、关键配置汇总（含本次新增）

> 完整列表见 `docs/功能测试清单.md` 与 `.env.local.example`，这里只列与本轮实现强相关的关键变量。

### 3.1 基础运行

- **NextAuth**  
  - `NEXTAUTH_SECRET`：使用 `openssl rand -base64 32` 生成。  
  - `NEXTAUTH_URL`：本地建议 `http://localhost:3001`（当前 `package.json` 中 dev 默认 `-p 3001`）。  
- **DeepSeek 解析**  
  - `DEEPSEEK_API_KEY`：用于 `/api/ai/parse-tasks` 与统一 AI 路由。  
  - 可选：`DEEPSEEK_MODEL`、`DEEPSEEK_BASE_URL`。  

### 3.2 推送与 Cron

- `NEXT_PUBLIC_VAPID_PUBLIC_KEY`、`VAPID_PRIVATE_KEY`：Web Push 所需；通过 `npx web-push generate-vapid-keys` 生成。  
- `CRON_SECRET`：定时任务密钥，Vercel Cron 等调用 `/api/cron/*` 时需在 Header 中携带。  

### 3.3 支付与订阅（Stripe）

- `.env.local.example` 新增条目：  
  - `STRIPE_SECRET_KEY=sk_xxx`  
  - `STRIPE_PRICE_ID_MONTHLY=price_xxx` （Pro 月付 Price ID）  
  - `STRIPE_PRICE_ID_YEARLY=price_xxx` （Pro 年付 Price ID）  
  - `STRIPE_WEBHOOK_SECRET=whsec_xxx` （Webhook 签名密钥，Endpoint 填 `/api/stripe/webhook`）  
- Next.js 端：  
  - `/pricing` 使用上述环境变量间接创建 Checkout Session；未配置时会返回「支付暂未配置」提示。  
  - `/api/stripe/webhook` 仅校验 Stripe 签名，不落地卡号。  

### 3.4 Skills / ClawHub / 搜索

- `CLAWHUB_API_BASE`：ClawHub 技能市场 API 基础地址；不填或设为 `https://clawhub.ai` 时启用；设为 `0` 则关闭。  
- `SERPER_API_KEY` 或 `BRAVE_API_KEY`：用于 `web_search` 与长期目标资料搜索。  
- `MCP_SERVER_URL` / `MCP_SERVER_HEADERS`：MCP Server 地址与额外头部，用于 `mcp_call` 技能。

## 四、功能与测试清单（本轮相关）

> 仅列本轮新增/受影响的功能与建议的手工测试步骤，完整覆盖见 `docs/功能测试清单.md`。

### 4.1 PMF 埋点与统计

1. 配置 `DEEPSEEK_API_KEY`、登录一个用户。  
2. 在 PandaChat 或 WBS 中触发一次自然语言解析（调用 `/api/ai/parse-tasks`）。  
3. 在 SQLite/Turso 中检查 `pmf_events` 表是否新增 `parse_tasks_success` 事件，`user_id` 为脱敏 ID。  
4. 参考 `docs/PMF-统计说明.md` 中 SQL 验证 Aha Moment 与日活统计是否可跑通。  

### 4.2 新客试用与配额

1. 注册新用户（邮箱或 Google），确认 `users.trial_until` 为 7 天后、`trial_count_used=0`。  
2. 触发多次解析/对话/冲突建议，观察 `trial_count_used` 递增，且在未超出 20 次前始终享受 Pro 配额。  
3. 当试用次数耗尽或试用期结束时，调用 AI 接口应返回统一的耗尽文案（`QUOTA_EXHAUSTED_MESSAGE`）。  

### 4.3 订阅与支付（需配置 Stripe 测试 Key）

1. 在 Stripe Dashboard 创建测试产品与月/年订阅 Price。  
2. 在 `.env.local` 中配置：`STRIPE_SECRET_KEY`、`STRIPE_PRICE_ID_MONTHLY`、`STRIPE_PRICE_ID_YEARLY`、`STRIPE_WEBHOOK_SECRET`。  
3. 进入 `/pricing`，点击「¥19.9 / 月」或「¥199 / 年」按钮：应跳转到 Stripe Checkout。  
4. 在 Stripe 测试模式下完成支付，确认 webhook 被调用，`users.subscription_status` 与 `subscription_plan` 更新为 `active/pro`，`subscription_end_date` 与 Stripe 的 `current_period_end` 一致。  

### 4.4 Skills 推荐与自主发现

1. 登录用户，创建若干任务：包含「开项目复盘会」「写周报」「订火车票去上海」等关键词。  
2. 调用 `GET /api/skills/recommend`：应返回如 `["deep_link_executor","message_draft","schedule_conflict"]` 等 ID。  
3. 在前端或调试中，根据推荐列表展示提示条或设置默认开启技能；后续可在此基础上接入 ClawHub/APM。  

### 4.5 Phase 7 多端（规划验证）

1. 阅读 `docs/Phase7-与远期规划.md`，确认其中：  
   - 技术选型（Uni-app/Taro）对比表；  
   - 与现有 API 的复用说明；  
   - 落地步骤 checklist；  
   - Zero-Shot Proactive V3 与自主技能发现的分工与扩展点。  
2. 若后续开启多端工程，可以此文档为 blueprint 开工，无需改动当前后端。
