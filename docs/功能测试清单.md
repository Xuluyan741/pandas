## 一、测试前准备

### 1.1 环境变量（按你要测的范围配）

| 用途 | 变量 | 必填？ | 说明 |
|------|------|--------|------|
| **对话 / 解析 / 冲突 / 长期目标** | `DEEPSEEK_API_KEY` | ✅ 必填 | 无则报错，对话不可用 |
| **网页搜索（搜一下、长期目标搜资料）** | `SERPER_API_KEY` 或 `BRAVE_API_KEY` | 可选 | 不配则搜索提示「未配置」 |
| **长期目标搜资料更丰富** | `SERPER_API_KEY` | 可选 | 不配则长期目标只给搜索引擎链接 |
| **社区技能发现/安装/执行** | `CLAWHUB_API_BASE` 不设为 `0` | 可选 | 设为 `0` 则关闭 ClawHub |
| **MCP 主对话内直接使用** | `MCP_SERVER_URL` | 可选 | 配了才能在主界面用 MCP 工具 |
| **登录（邮箱/Google）** | `NEXTAUTH_SECRET`、`NEXTAUTH_URL`；Google 需 `GOOGLE_CLIENT_ID/SECRET` | 可选 | 不配只能演示模式 |
| **推送** | `NEXT_PUBLIC_VAPID_PUBLIC_KEY`、`VAPID_PRIVATE_KEY`、`CRON_SECRET` | 可选 | 不配收不到推送 |

**除 `DEEPSEEK_API_KEY` 外，想「全部功能都用」还需要配：**

| 功能 | 需要配置的变量 | 获取/说明 |
|------|----------------|-----------|
| **登录（邮箱 + Google）** | `NEXTAUTH_SECRET`、`NEXTAUTH_URL`（本地如 `http://localhost:3001`） | `openssl rand -base64 32` 生成 secret |
| | Google 登录：`GOOGLE_CLIENT_ID`、`GOOGLE_CLIENT_SECRET`，并设 `NEXT_PUBLIC_GOOGLE_ENABLED=true` | [Google Cloud Console](https://console.cloud.google.com) → 凭据 → OAuth 2.0 客户端 ID，回调 URI 填 `http://localhost:3001/api/auth/callback/google` |
| **网页搜索** | `SERPER_API_KEY` 或 `BRAVE_API_KEY`（二选一即可） | [serper.dev](https://serper.dev) 或 Brave Search API |
| **社区技能** | `CLAWHUB_API_BASE=https://clawhub.ai`（或不填，默认即此；不要设为 `0`） | 无需 key，保持启用即可 |
| **MCP** | `MCP_SERVER_URL`（你的 MCP 服务 HTTP 地址）；可选 `MCP_SERVER_HEADERS` | 自建或使用提供 HTTP 的 MCP 网关 |
| **推送** | `NEXT_PUBLIC_VAPID_PUBLIC_KEY`、`VAPID_PRIVATE_KEY`；定时任务还需 `CRON_SECRET` | `npx web-push generate-vapid-keys`；`openssl rand -base64 24` 生成 CRON_SECRET |
| **登录后数据存云端** | `TURSO_DATABASE_URL`、`TURSO_AUTH_TOKEN` | [Turso](https://turso.tech) 建库拿 URL 和 token |
| **Pro 订阅支付** | `STRIPE_SECRET_KEY`、`STRIPE_PRICE_ID_MONTHLY`、`STRIPE_PRICE_ID_YEARLY`、`STRIPE_WEBHOOK_SECRET` | [Stripe Dashboard](https://dashboard.stripe.com) |
| **云端语音转写** | `OPENAI_BASE_URL`、`OPENAI_API_KEY`（或 Whisper 兼容接口） | 若用 gptsapi 等，填对应地址和 key |

## 二、主界面 · 对话（和 small熊猫说话）

### 2.1 日程解析与写入（无冲突）

| 前置 | 操作 | 预期 |
|------|------|------|
| 至少有一个项目（没有则在工作台 → 添加 → 新建项目） | 输入「明天上午 10 点写周报」并发送 | 小熊猫回复 + 任务卡片；工作台 → 任务列表里**多一条**「写周报」，无需再点 |
| 同上 | 输入「这周五下午开会」并发送 | 解析出对应日期的一条任务并写入 |

### 2.2 日程解析 + 冲突检测（需再点一步）

| 前置 | 操作 | 预期 |
|------|------|------|
| 已有任务占用了「明天下午 3 点」 | 输入「明天下午三点和产品开会」并发送 | 出现「检测到日程冲突」+ 建议（移动/顺延等）；点「好的」按建议写入，或「暂不」直接写入 |
| 同上 | 点「好的」 | 已有任务时间被调整（若有建议）+ 新任务写入 |

### 2.3 长期目标（规划 + 一键写入）

| 前置 | 操作 | 预期 |
|------|------|------|
| 无 | 输入「帮我规划五一去韩国旅游」并发送 | 出现「长期目标」卡片：分步计划（日期、优先级）+ 推荐资料（配了 Serper 才有真实摘要） |
| 上一步已出现计划卡片 | 点击「一键写入到日历」 | 工作台 → 大盘出现该长期目标；任务列表里出现多条子任务，带「长期目标」标识和资料链接 |
| 已有长期目标 | 工作台 → 大盘 → 长期目标卡片 | 可「暂停/恢复/完成/移除」；今日任务、进度条、剩余天数正常 |

### 2.4 网页搜索

| 前置 | 操作 | 预期 |
|------|------|------|
| `.env.local` 已配 `SERPER_API_KEY` 或 `BRAVE_API_KEY` | 输入「搜一下 让子弹飞 PPT 模板」并发送 | 回复中带「【网页搜索】」+ 若干条标题/链接/摘要 |
| 未配上述 key | 同上 | 回复中带「【网页搜索】暂时无法联网搜索（未配置…）」 |

### 2.5 Deep Link（打车/订餐/买票等）

| 前置 | 操作 | 预期 |
|------|------|------|
| 无 | 输入「帮我打个车去机场」并发送 | 出现「执行预览卡」（如高德打车），点「允许并打开」可跳转（不自动下单） |
| 无 | 输入「点个外卖」或「下周一下午 2 点高铁去上海」等 | 对应美团/淘宝/12306 等预览卡（若有配置） |

### 2.6 带时间 + 要完成的事（fallback 写入）

| 前置 | 操作 | 预期 |
|------|------|------|
| 无 | 输入「明天晚上要完成 PPT」并发送 | 至少出现一条任务建议（标题如「完成 PPT」、明天日期）；无冲突则直接写入，有冲突则要点「好的/暂不」 |

## 九、最小可测集合（只配 DEEPSEEK_API_KEY）

若只配了 `DEEPSEEK_API_KEY`，至少可验证：

1. **主对话**：说「明天上午写周报」→ 任务列表多一条。
2. **冲突**：先加一条「明天下午 3 点开会」，再说「明天下午三点和产品复盘」→ 出现冲突建议，点「好的」或「暂不」。
3. **长期目标**：说「帮我规划五一去韩国旅游」→ 出现计划卡片 + 一键写入；大盘里能看到该目标。
4. **Deep Link**：说「帮我打个车」→ 出现预览卡（点不点开都可）。
5. **工作台**：打开悬浮球，切大盘/任务/日历/甘特/添加，界面正常。

网页搜索、社区技能、MCP、推送需按上表补全对应环境变量后再测。
